# OGC Mechanics Adaptation (Starting Point)

We will use **OGC** as the starting mechanical reference for:

- classes
- abilities
- baseline stat interactions

## Adaptation Approach

- Start close to OGC semantics for speed
- Rename/re-theme where needed for Sigil/Bastion fiction
- Keep combat timing and effect ordering explicit and testable
- Maintain a changelog of divergences from OGC baseline

## Guardrails

- Avoid speculative complexity in v1
- Prefer “boring clear rules” over clever ambiguity
- Every ability should declare:
  - target rules
  - range/LOS
  - resource cost
  - timing window
  - deterministic resolution order
